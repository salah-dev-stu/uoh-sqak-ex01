## 1. Theoretical Background

A vanilla RNN processes the 10-sample context window $x \in \mathbb{R}^{10}$ as a sequence of timesteps $t = 1 \ldots 10$. At each step it consumes a 5-dim input vector $\tilde{x}_t = [x_t,\, C]$ (one signal sample concatenated with the 4-dim one-hot $C$ — see ADR-003) and updates a hidden state:

$$
h_t = \tanh\bigl(W_{xh}\,\tilde{x}_t + W_{hh}\,h_{t-1} + b_h\bigr)
$$

with $h_0 = \mathbf{0}$. After the final step we project the last hidden state to the output:

$$
\hat{y} = W_{ho}\,h_{10} + b_o
$$

We use `torch.nn.RNN(input_size=5, hidden_size=H, num_layers=L, nonlinearity='tanh', batch_first=True, dropout=p)` followed by a `nn.Linear(H, 10)`.

### 1.1 Why concat C at every timestep (ADR-003)
Three options exist:
1. Use $C$ to initialize $h_0$.
2. Use $C$ as the input only at $t=0$.
3. Concatenate $C$ to the input at **every** timestep.

We chose (3) because:
- The selector $C$ is the "task instruction" — it must remain present in case the network "forgets" it (vanilla RNNs are notorious for this).
- Symmetric with LSTM (also gets $C$ at every step), so any architectural difference traces to the *cell*, not the conditioning.
- Simple to reason about and test.

### 1.2 The lecturer's hypothesis (H1)
**RNN excels at high frequencies (100, 200 Hz)** because the rapid oscillation completes whole cycles within the 10-sample window: $100\,\text{Hz} \cdot \tfrac{10}{1000\,\text{Hz}} = 1.0$ cycle per window, and $200\,\text{Hz} \to 2.0$ cycles. The 20 / 60 Hz channels sit in the sub-cycle regime (0.2 / 0.6 cycles), which is why H2 hands those to LSTM. The vanilla RNN's tanh recurrence has a fair shot at the multi-cycle regime: each 10-step unfold sees a full sinusoidal swing, so the network can in principle learn a tight phase tracker without needing the LSTM's gating overhead.

The empirical verdict — see `notebooks/analysis.ipynb` §7.5 — is that LSTM still wins at high freq (RNN mean MSE 4.87 × 10⁻⁴ vs LSTM 1.67 × 10⁻⁴, p = 1.2e-07). The most likely mechanism is that tanh's gradient pathology is regime-blind once recurrence has signal to grip onto; LSTM's cell-state highway sidesteps this. **The frequency design makes the test meaningful** — H1 is *meaningfully* disconfirmed rather than precluded by sub-cycle inputs.

### 1.3 Parameter count
For `nn.RNN(input=5, hidden=H, layers=1)`:
$$
P_{\text{cell}} = H \cdot (5 + H) + H = H \cdot (H + 5) + H
$$
plus output linear $H \cdot 10 + 10$. For $H=128$: $P = 128 \cdot 133 + 128 + 1\,290 = 17\,034 + 1\,290 = 18\,324$.

## 2. Inputs

### 2.1 Constructor (`__init__`)
| Param | Type | Default | Range |
|---|---|---|---|
| `input_dim_per_step` | int | `5` | == 5 (sample + 4 one-hot) |
| `output_dim` | int | `10` | == 10 |
| `hidden_size` | int | `128` | > 0 |
| `num_layers` | int | `1` | 1, 2, or 3 |
| `dropout` | float | `0.0` | [0.0, 1.0); only active when num_layers > 1 |

### 2.2 Forward (`forward(x)`)
| Tensor | Shape | Dtype |
|---|---|---|
| `x` | `(batch_size, 10, 5)` | `float32` |

Each timestep's 5-dim input is `[sample_t, C[0], C[1], C[2], C[3]]`.

---

## 3. Outputs

| Tensor | Shape | Dtype |
|---|---|---|
| `y_hat` | `(batch_size, 10)` | `float32` |

The 10-dim prediction = the inferred clean window of the selected sine. Generated by reading the **last** hidden state and projecting to 10 outputs.

## 6. Constraints

- **Hard**: file ≤ 150 LoC (RULES.md §8). Use a `_helpers.py` if needed.
- **Hard**: nonlinearity must be `tanh` (per RNN book + IDEA.md).
- **Hard**: input dim per step **must** equal 5; output dim **must** equal 10.
- **Soft**: hidden_size ≤ 256, num_layers ≤ 3.
- **Soft**: dropout > 0 only meaningful when num_layers > 1 (`nn.RNN` ignores otherwise; we accept the default 0.0).

## 7. Alternatives Considered

| Option | Chosen? | Reason |
|---|---|---|
| Tanh nonlinearity | Yes | RNN book + lecturer requirement |
| ReLU nonlinearity | No | Causes exploding activations in vanilla RNNs |
| C as init hidden state | No (ADR-003) | Conditioning fades; less robust |
| C only at t=0 | No (ADR-003) | Same |
| C at every timestep | Yes (ADR-003) | Robust, symmetric with LSTM |
| Bidirectional RNN | No | Out of scope; `IDEA.md` specifies vanilla |
| Use mean pooled hidden states | No | Last-step is conventional; simpler to reason about |
| Use all 10 hidden states → linear (10×H, 10) | No | Doubles params; minor improvement at best |
| GRU instead of RNN | No | Out of scope (GRU = registry stub for extension demo) |

## 8. Success Criteria & Test Scenarios

### 8.1 Success criteria
1. `RNNExtractor(...).forward(x)` returns `(B, 10)` for any `B ≥ 1`, `x` of shape `(B, 10, 5)`.
2. `count_parameters()` matches the formula in §1.3.
3. Tanh activation is in use (introspect `module.nonlinearity == 'tanh'`).
4. Save/load round-trip is bitwise identical.
5. Training 1 epoch on a memorizable subset reduces train loss.
6. Hidden state initialized to zeros at every forward call (no carryover between batches).

### 8.2 Test scenarios
Per TODO Phase 6c (RNN-001..040): shape tests, param count, save/load, dropout active in train mode, deterministic in eval mode, gradient flow, integration with registry, smoke training.

### 8.3 Failure modes
- `num_layers ∉ {1, 2, 3}` → `ValueError`
- `hidden_size ≤ 0` → `ValueError`
- `dropout ∉ [0, 1)` → `ValueError`
- Input shape != `(B, 10, 5)` → upstream `RuntimeError` from `nn.RNN`
- NaN gradient (rare with tanh) → caught by training loop, run aborted
